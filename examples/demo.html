How it works

    
    
      
  1. Your prompt arrives at the proxy with actor + destination metadata.
    2. 
      
  2. A classifier tags spans (PHI, PII, secrets) using a YAML taxonomy.
    3. 
      
  3. The policy engine matches tagged spans against your rules → verdict (allow / redact / block).
    4. 
      
  4. The proxy calls the LLM (or doesn't, if blocked), then writes a content-hashed receipt to disk. Receipt path is shown above.
    5. 
    


    
What's in a receipt

    
A JSON record with: actor, destination, classifier tags, policy set version, verdict, redactions applied, prompt + response text and hashes, timestamp. Blocked prompts are stored in the receipt so an auditor can see what was attempted; retention policy protects that audit log over time.
