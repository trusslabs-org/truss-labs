Truss Audit Proxy

Truss sits between an agent and an LLM. Every call passes through a YAML policy that can allow, redact, or block — and every call writes a signed receipt to disk. The receipt is what your auditor reads, not the chat log.

Flow: prompt → classifier → policy → LLM → receipt. The verdict and receipt path below are produced server-side from your YAML rules.

What's in a receipt

    
A JSON record with: actor, destination, classifier tags, policy set version, verdict, redactions applied, prompt + response hashes, timestamp. No prompt or response body — the receipt is for audit, not replay.
